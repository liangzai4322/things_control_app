# TaskBox 核心功能与业务规则

最后核对：2026-07-16。

TaskBox 是一套以“盒子”为入口的个人行动系统。核心闭环是：任务进入合适盒子，按日期与主线推进，完成后进入可追溯历史并结算积分，积分再用于兑换奖励；放松池、清单、小世界和弑神塔承担不同于待办的用途。

## 1. 首页与盒子

- 首页显示重要、待办、放松、思路等核心盒子，也允许创建自定义盒子。
- 重要盒和待办盒固定在前两位；其他盒子可删除、排序或置顶。
- 手机保持双列信息密度，桌面端利用更宽视口展示更多盒子和任务信息。
- 核心盒导航在首页、盒子详情、积分、小世界和主线页保持一致；详情页不重复显示当前盒子。
- 每个盒子有独立名称、主题色、图标、类型、介绍文案和“每日一句”。每日一句可直接点击编辑并同步到服务器。
- 首页提供日期视图，集中查看今日新增、计划任务、逾期任务和已完成任务。

## 2. 盒子类型

盒子不是都等于待办。创建后仍可修改类型，但修改前需要确认。

| 类型 | 用途 | 核心行为 |
| --- | --- | --- |
| 任务盒 | 重要、待办等需要推进和完成的事项 | 计划、截止、进度、完成、积分、周期任务、置顶 |
| 放松池 | 放松活动、奖励候选、惩罚候选 | 抽取、使用记录、冷却时间、积分成本 |
| 清单盒 | 灵感、资料、备忘、长期收藏 | 收藏、标签、链接、归档，不计入待办完成率 |

思路盒内容可以移动到重要盒或待办盒；重要盒和待办盒之间也可以通过任务菜单互相迁移。

## 3. 任务生命周期

- 任务字段包括内容、盒子、优先级、计划时间、截止时间、积分、进度、置顶档位、备注、主线和里程碑关联。
- 执行方式独立于设备场景，分为“我来做”“交给 AI”“人机协作”。AI 任务仍可设置在哪个设备查看，列表使用独立标签标识，周期实例继承执行方式。
- 任务可标记为电脑、手机或通用。旧任务迁移为通用，新任务默认电脑；自动场景优先显示当前设备任务，也可手动切换为手机、电脑或全部。
- 非当前设备任务不被删除或彻底隐藏，在盒子详情的“其他设备任务”折叠区中保留访问入口。
- 跨天任务可从右键或移动端三点菜单选择“今天收工”，保存当前进度、续接备注和再次出现时间；默认次日 08:00 回到列表。
- 单次任务计划到未来日期时，盒子当前任务区不会提前显示；它在计划日 00:00 自动释放，在此之前可从“待到点出现”和日期页对应日期查看、编辑。
- “今日已收工”默认折叠，可提前选择“继续显示”；如果再次出现时间晚于截止时间，保存前会明确提示。
- 截止时间精确到小时和分钟；到期后整条任务进入危险红色状态。
- 重要盒置顶只有第一、第二、第三三个档位，右键菜单通过可视化选择器设置。
- 任务支持右键菜单和移动端操作入口，可编辑、删除、置顶、迁移盒子。
- 删除进入默认折叠的“已删除”，不会立即物理移除；可还原误删内容。
- 已完成按时间倒序分组：一周内按天，一周以外归入“一周以前”，越早完成越靠下。
- 完成和删除使用稳定记录 ID；同一条任务的重复视图不得产生重复记录或级联误删。
- 保存编辑支持 `Ctrl+Enter`。

## 4. 日期与周期任务

- 日期页提供今天、今晚、明天等快捷计划与截止时间。
- “今天”的计划时间按当前时间向上取整后加两小时；“今天截止”默认当日 22:00，“今晚截止”默认次日 00:00。
- 周期支持每天、间隔若干天、每周、每月，并可暂停、修改整个系列或删除整个系列。
- 周期模板不作为普通任务显示；系统以 `recurrenceKey` 保证同一周期实例唯一。
- 每个周期规则有独立出现时间，默认 08:00。完成当前实例后可以预生成下一实例，但在 `visibleAfter` 到达前只进入“待到点出现”，不会混入盒子当前任务和今日统计；日期页对应日期仍可提前查看。
- 完成周期实例后，下一实例按规则生成，完成历史仍保留；每天、间隔日、每周和每月共用同一释放机制。

## 5. 主线与里程碑

- 同时推进的活跃主线建议控制在 2 至 6 条，系统上限为 6 条；最多两条可标记“本周重点”。
- 主线记录最终成果、当前阶段、状态、目标日期、识别色和排序；创建时提供 8 组独立于盒子主题色的矿物色识别方案。
- 每条主线可拆为多个里程碑，任务可关联主线与具体里程碑。
- 首页显示主线概览；`#mainline/:id` 页面显示下一行动、里程碑和关联任务。
- 删除主线会删除其里程碑，并解除任务关联，不删除任务本身。

## 6. 积分与奖励

- 积分账户由流水计算，不把余额当作不可追溯的独立数字。
- 完成任务自动产生收入流水；取消完成或调整任务分值时会对账修正。
- 支持历史积分补录、手动记账、AI 文本识别后人工确认入账。
- 奖励池支持新增、修改、启停和兑换；兑换成功产生支出流水。
- 积分页面展示余额、近期变化、规则、奖励和流水。
- 积分数据通过同一服务器 API 读写，失败时先保留本地数据并在后续操作重试。

## 7. 小世界

- 珍宝阁和弑神塔是独立的游戏化内容库，不算待办任务。
- 两者按层级和记录读取，支持新增、修改、删除和完成状态。
- 弑神塔可快速跳层；抽奖转盘显示候选内容和结果。
- 外部服务配置集中在小世界设置中。

## 8. 同步、离线与认证

- 唯一正式云端数据源是自建服务器上的 SQLite API，Gist 已退出运行链路。
- 页面启动时用本地缓存快速首屏，再拉取服务器快照并立即刷新当前路由。
- 新增、修改、删除按记录调用 API，不上传整份 JSON。
- API Token 仅保存在当前浏览器本地；填写后立即拉取盒子、积分和小世界数据。
- 设备场景偏好同样是当前浏览器的本地设置，手机和电脑可以分别保持自己的模式。
- 云端请求失败时保留本地缓存和待同步操作，后续写操作继续重试。
- 同步提示只在远端内容确有变化或写入成功时出现，不能每次刷新都误报。

## 9. 数据安全与恢复

- 线上只部署 `dist/` 压缩产物，不部署源码映射。
- Token、服务器密码、Cookie 和原始认证头不得写入仓库、文档、日志或静态构建。
- 前端发布前保留 Git 标签或本地压缩备份；服务器变更前备份数据库和 API 目录。
- SQLite 开启 WAL；服务端写入以记录 ID 幂等更新，避免整文件覆盖和重复导入。

实现细节见 `docs/architecture.md`，部署与恢复见 `docs/runbook.md`。
